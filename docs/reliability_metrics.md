# 编码者信度指标说明

本文档说明 ITIAS Coder「编码者信度」功能输出的指标含义、计算方法、适用场景与局限。

## 功能概述

当两名编码者对**同一组课堂切片**分别完成 ITIAS 编码并导出 Excel 后，可将两个文件导入本工具，自动对齐片段序号并计算一致性指标，生成 Excel 报告。

## 输入数据格式

复用标准 ITIAS 编码导出格式（与「导入 Excel 续作」相同）：

| 列 | 字段 | 说明 |
|----|------|------|
| A | 片段序号 | 1-based 整数 |
| C | 编码 | ITIAS 代码 ID（1–18），空表示未编码 |

两名编码者的导出文件应来自**同一套切片**（片段序号一一对应）。

## 输出指标

### 1. 百分比一致性（Percentage Agreement）

**定义**：两名编码者在同一片段上给出相同 `code_id`（含双方均未编码）的片段数，占总可比片段数的比例。

```
一致性(%) = 一致片段数 / 总片段数 × 100
```

报告同时提供：

- **全部片段**：包含任一方未编码的片段（双方均为空也算一致）。
- **仅双编码片段**：仅统计两名编码者均已有编码的片段。

**解读**：直观反映编码吻合程度。课堂互动研究中常用于初步筛查编码质量；但高一致性也可能仅因某些高频行为（如「沉寂」）占主导。

### 2. Cohen's Kappa（κ）

**定义**：在剔除偶然一致后的名义量表一致性系数。取值范围通常为 −1 到 1：

| κ 值 | 一般解读 |
|------|----------|
| < 0.20 | 轻微 |
| 0.21–0.40 | 一般 |
| 0.41–0.60 | 中等 |
| 0.61–0.80 | 较好 |
| > 0.80 | 很好 |

**计算**：设观察一致比例 \(p_o\)，期望偶然一致比例 \(p_e\)（基于边际分布），则

\[
\kappa = \frac{p_o - p_e}{1 - p_e}
\]

未编码片段在计算中作为独立类别（类别 0）参与；若需仅考察已编码片段，请参考「仅双编码」百分比一致性。

**解读**：比单纯百分比一致性更能排除「碰巧一致」的影响，是课堂观察编码研究中的常用指标。

### 3. 混淆矩阵（Confusion Matrix）

18×18 矩阵：行 = 编码者 1 的代码，列 = 编码者 2 的代码，单元格 = 该组合的片段数。**仅统计双方代码均在 1–18 范围内的片段**。

用于定位系统性分歧（例如编码者 1 常用「讲授(6)」而编码者 2 常用「提问(5)」）。

### 4. 分编码一致性（Per-code Agreement）

对每个 ITIAS 代码（1–18），统计：

- 各编码者单独使用该代码的次数；
- 双方同时使用该代码的次数；
- **一致率** = 双方一致次数 / 至少一方使用该代码的片段数。

用于发现特定行为类别的一致性问题。

## 适用场景

- 编码培训阶段：两名编码者试编码同一节课，评估是否达到可独立编码的标准（常参考 κ ≥ 0.80）。
- 研究项目中期核查：主研与助理编码者定期比对，发现编码漂移。
- CCIES 工作流对标：CCIES 支持双编码者信度分析；本功能提供同类核心指标（百分比一致性、Kappa、混淆矩阵）。

## 局限与注意事项

1. **样本量**：片段数过少（如 < 30）时，κ 和标准误不稳定，解读应谨慎。
2. **Kappa 悖论**：在极度不平衡的类别分布下（如大量「沉寂」），κ 可能低于百分比一致性所暗示的水平；应结合混淆矩阵与分编码统计综合判断。
3. **未编码片段**：若一方大量未编码，会拉低整体一致性；建议先完成双编码再计算，或重点关注「仅双编码」指标。
4. **片段对齐**：工具按片段序号对齐，不校验视频文件名；请确保两份导出来自同一切片集。
5. **非 F 检验**：本实现输出 Kappa 与混淆矩阵等 CCIES 常用信度指标；未包含 ANOVA F 检验。若研究设计需要 F 检验，请在 SPSS 等统计软件中基于导出的混淆矩阵或原始编码序列另行计算。
6. **编码框架**：默认使用 ITIAS 18 类框架（顾小清 & 王炜, 2004）；混淆矩阵行列标签与此框架一致。

## 使用入口

- **图形界面**：主界面「第四步：编码者信度」— 选择两名编码者的 Excel 导出，点击「计算信度」。
- **命令行**：

  ```bash
  python -m itias_coder --reliability coder1.xlsx coder2.xlsx [output.xlsx]
  ```

  未指定输出路径时，默认在第一个文件同目录生成 `{文件名}_reliability.xlsx`。
